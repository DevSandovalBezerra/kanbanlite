# Registro de Alterações — 06/04/2026

## Resumo
- Correção do fluxo “Nova Coluna” no board: o card agora dispara criação via API.
- Renomear colunas no Kanban com duplo clique (UI) + endpoint `PATCH /api/columns`.
- Ajuste do seed local: criador do projeto passa a ser inserido como `owner` em `project_members`.

## Kanban — Criar Coluna
- UI “Nova Coluna” passou a ter handler no JavaScript e envia:
  - `POST /api/columns` — body: `{ board_id, name, position }`
- Após sucesso, a página recarrega para refletir a nova coluna.

## Kanban — Renomear Coluna
- Duplo clique no nome da coluna (ou Enter com foco) abre prompt para renomear.
- API:
  - `PATCH /api/columns` — body: `{ id, name }`
- Regras:
  - Mantida a mesma política de permissão de board (manager+).
  - Validação de nome (obrigatório, máx. 60 chars) e mensagem amigável em caso de duplicidade.

## Seed Local
- O seed adiciona o criador do projeto como membro `owner` para que a listagem de projetos reflita as regras de visibilidade por membro.
