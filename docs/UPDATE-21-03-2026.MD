## Segurança Aplicada

| Medida | Descrição |
|--------|-----------|
| S01 | `resolveProjectId()` nos repositories — previne spoofing de `project_id` no request |
| S02 | Endpoint PATCH em `/api/admin/users?id=X` com validação de campos |
| S03 | `AdminMiddleware` revalida `is_admin` no banco — revogação de privilégios tem efeito imediato |
| S04 | Endpoint dedicado `reset-password` separado do UPDATE de usuário |
| S05 | Bloqueio de soft-delete de `owner` com projetos ativos; retorna lista dos projetos afetados |
| S06 | Política de senha: mínimo 8 chars, 1 maiúscula, 1 número (`/^(?=.*[A-Z])(?=.*\d).{8,}$/`) |
| S07 | `admin_audit_log` — toda ação administrativa gera registro auditável |
| S08 | Duplo guard sequencial nos controllers: (1) tenant isolation → 404; (2) papel → 403 |
| S09 | `AddMemberService` valida que o `user_id` pertence à mesma empresa da sessão |
| S10 | `LEFT JOIN users` no `findByColumnId()` — tarefas com responsável deletado retornam `null` sem erro |
| S11 | Bloqueio de remoção/desativação do último admin da empresa |
| S12 | Somente `owner` pode alterar o papel de membros existentes (PATCH) |

## Novos Endpoints da API

### Usuários (requer AdminMiddleware)
| Método | Rota | Ação |
|--------|------|------|
| GET | `/api/admin/users` | Lista usuários da empresa |
| POST | `/api/admin/users` | Cria novo usuário |
| PATCH | `/api/admin/users?id=X` | Atualiza nome, e-mail, is_admin |
| POST | `/api/admin/users/toggle-status` | Ativa ou desativa usuário |
| POST | `/api/admin/users/reset-password` | Redefine senha |
| DELETE | `/api/admin/users?id=X` | Soft-delete de usuário |

### Membros do Projeto
| Método | Rota | Permissão |
|--------|------|-----------|
| GET | `/api/project-members?project_id=X` | Qualquer membro |
| POST | `/api/project-members` | `owner` ou `manager` |
| PATCH | `/api/project-members?id=X` | `owner` apenas |
| DELETE | `/api/project-members?id=X` | `owner` ou próprio usuário |
