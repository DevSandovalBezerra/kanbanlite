# Registro de Alterações — 27/03/2026

## Resumo
- Implementado o menu Secrets por projeto, com compartilhamento seguro de chaves/senhas visíveis aos membros do projeto.
- Restrições de visibilidade de projetos: usuários só veem/acessam projetos onde são membros (mesma empresa não é suficiente).
- Correção do caminho de avatar (404): normalização do URL considerando aplicações servidas em subpasta (ex.: /kanban/index.php).

## Novidades — Secrets por Projeto
- Página de gerenciamento: `/projects/secrets?id={project_id}`.
- Acesso:
  - Listar/visualizar: qualquer membro do projeto (viewer+).
  - Criar/editar/excluir: editor, manager ou owner.
- Segurança:
  - Valores de secrets armazenados criptografados (AES-256-GCM) com chave derivada de `APP_SECRET`.
  - Na UI, os valores são visíveis aos membros (objetivo: evitar troca via messengers).
  - Botão de copiar valor na lista.
- API:
  - `GET /api/project-secrets?project_id={id}`
  - `POST /api/project-secrets` — body: `{ project_id, secret_key, secret_value }`
  - `PATCH /api/project-secrets?id={secretId}` — body: `{ secret_key, secret_value }`
  - `DELETE /api/project-secrets?id={secretId}`

## Restrições de Acesso a Projetos

- Acesso direto por URL a boards/projetos fora do convite retorna 404/403 conforme o caso.
